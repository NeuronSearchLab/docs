Routes that govern how candidates are retrieved, scored, and reordered before being served. Background and conceptual overviews live in the [Contexts](/guides/contexts), [Rules Engine](/guides/rules-engine), [Pipeline Config](/guides/pipeline-config), and [A/B Testing](/guides/ab-testing) guides.

## Contexts

### List contexts

```http
GET /api/context
```

<CodeGroup>

```bash cURL
curl "https://console.neuronsearchlab.com/api/context" \
  -H "Authorization: Bearer nsl_<prefix>_<token>"
```

</CodeGroup>

### Create context

```http
POST /api/context/create
```

<CodeGroup>

```bash cURL
curl -X POST https://console.neuronsearchlab.com/api/context/create \
  -H "Authorization: Bearer nsl_<prefix>_<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Home Feed",
    "description": "Logged-in homepage rail",
    "json_data": { "exclude_viewed_items": true }
  }'
```

</CodeGroup>

### Update context

```http
PATCH /api/context/{id}
```

<CodeGroup>

```bash cURL
curl -X PATCH https://console.neuronsearchlab.com/api/context/101 \
  -H "Authorization: Bearer nsl_<prefix>_<token>" \
  -H "Content-Type: application/json" \
  -d '{ "json_data": { "exclude_viewed_items": false } }'
```

</CodeGroup>

### Resolve active model

```http
GET /api/context/{id}/active-model
```

Returns the model version currently serving traffic for this context.

<CodeGroup>

```bash cURL
curl "https://console.neuronsearchlab.com/api/context/101/active-model" \
  -H "Authorization: Bearer nsl_<prefix>_<token>"
```

</CodeGroup>
