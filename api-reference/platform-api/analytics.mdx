Read-only routes that power the console's reporting and debugging surfaces. See the [Analytics guide](/guides/analytics) and the [Explainability guide](/guides/explainability) for context.

## Dashboard

### Dashboard metrics

```http
GET /api/dashboard
```

<CodeGroup>

```bash cURL
curl "https://console.neuronsearchlab.com/api/dashboard" \
  -H "Authorization: Bearer nsl_<prefix>_<token>"
```

</CodeGroup>

## Analytics

### Recommendation, event, and catalog analytics

```http
GET /api/analytics
```

Accepts `from`, `to`, `granularity` (`hour` / `day` / `week`), and optional `context_id` query parameters.

<CodeGroup>

```bash cURL
curl "https://console.neuronsearchlab.com/api/analytics?from=2026-04-01&to=2026-04-29&granularity=day" \
  -H "Authorization: Bearer nsl_<prefix>_<token>"
```

</CodeGroup>

### Compare items

```http
POST /api/analytics/items/compare
```

<CodeGroup>

```bash cURL
curl -X POST https://console.neuronsearchlab.com/api/analytics/items/compare \
  -H "Authorization: Bearer nsl_<prefix>_<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "item_ids": ["itm_7f3a2c9e", "itm_a8d1b247"],
    "from": "2026-04-01",
    "to": "2026-04-29"
  }'
```

</CodeGroup>

## Search and explainability

### Console-wide search

```http
GET /api/search
```

<CodeGroup>

```bash cURL
curl "https://console.neuronsearchlab.com/api/search?query=home%20feed" \
  -H "Authorization: Bearer nsl_<prefix>_<token>"
```

</CodeGroup>

### Explain a recommendation

```http
GET /api/explain
```

Pass either `request_id` (preferred) or `user_id` + `context_id` + `item_id` to inspect why an item ranked where it did.

<CodeGroup>

```bash cURL
curl "https://console.neuronsearchlab.com/api/explain?request_id=c7f0d2bb-56b1-4ca7-8f09-84d8e4ed02b4&item_id=itm_7f3a2c9e" \
  -H "Authorization: Bearer nsl_<prefix>_<token>"
```

</CodeGroup>

POST /api/assistant/chat
